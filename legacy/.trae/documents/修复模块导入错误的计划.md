# 修复模块导入错误的计划

## 问题分析
从错误信息和代码检查中，我发现了两个问题：

1. **主要错误**：在 `src/common/__init__.py` 文件的 `__all__` 列表中包含了 `"decompile_utils"`，但实际上这个模块并没有被定义或导入
2. **次要问题**：`logger_utils` 模块在同一个文件中被重复导入了两次

## 修复方案

### 1. 修复 src/common/__init__.py 文件

**问题1：移除不存在的 decompile_utils**
- 将第140行 `"decompile_utils"` 从 `__all__` 列表中删除
- 实际的反编译功能是通过 `jar_utils` 模块提供的

**问题2：移除重复的 logger_utils 导入**
- 保留第75-92行的 `logger_utils` 导入
- 删除第93-112行重复的 `logger_utils` 导入

### 2. 验证修复效果

修复后，运行 `src/main.py` 来验证导入错误是否已经解决。

## 预期效果

- 修复后，`src/main.py` 应该能够正常导入所有模块
- 不再出现 `AttributeError: module 'src.common' has no attribute 'decompile_utils'` 错误
- 代码结构更加清晰，没有冗余的导入

## 修复步骤

1. 编辑 `src/common/__init__.py` 文件
2. 删除第140行的 `"decompile_utils"`
3. 删除第93-112行重复的 `logger_utils` 导入
4. 保存文件
5. 运行 `src/main.py` 验证修复效果
