# Localization\_File目录结构测试计划

## 1. 测试目标

验证Localization\_File目录结构的完整性、权限和可用性，确保本地化工具能够正常运行。

## 2. 测试内容

### 2.1 目录结构验证

* 检查Localization\_File根目录存在性

* 验证所有必要子目录存在性

* 测试目录层级结构正确性

### 2.2 权限测试

* 测试所有目录的读写权限

* 验证文件创建、读取和删除操作

* 测试临时文件处理能力

### 2.3 测试场景

1. **源代码目录测试**

   * `source/English/mod_name/src/`

   * `source/Chinese/mod_name/src/`

   * `source_backup/English/mod_name/src/`

   * `source_backup/Chinese/mod_name/src/`

2. **JAR文件测试**

   * `source/English/mod_name/mod.jar`(目录权限)

   * `source/Chinese/mod_name/mod.jar`(目录权限)

   * `source_backup/English/mod_name/mod.jar`(目录权限)

   * `source_backup/Chinese/mod_name/mod.jar`(目录权限)

3. **JAR反编译输出目录测试**

   * `source/English/mod_name/jar/`

   * `source/Chinese/mod_name/jar/`

   * `source_backup/English/mod_name/jar/`

   * `source_backup/Chinese/mod_name/jar/`

4. **规则文件目录测试**

   * `rule/English/mod_name/`

   * `rule/Chinese/mod_name/`

5. **输出目录测试**

   * `output/Extend_en2zh/mod_name/`

   * `output/Extend_zh2en/mod_name/`

## 3. 测试工具

使用已创建的测试脚本 `test_localization_file.py` 执行测试，该脚本具有以下功能：

* 自动检测目录结构

* 测试目录读写权限

* 验证文件操作能力

* 生成详细测试报告

## 4. 执行步骤

1. 运行测试脚本：`python src/test_localization_file.py`
2. 等待测试完成
3. 查看控制台输出的测试结果
4. 检查生成的测试报告文件

## 5. 预期输出

* 控制台显示测试结果摘要和详细信息

* 生成测试报告文件 `D:\Poki\Tool\Localization_File\localization_file_test_report.txt`

* 所有测试项显示通过状态(PASS)

## 6. 后续处理

根据测试结果，对失败项进行分析和修复：

* 目录缺失：创建相应目录

* 权限问题：调整目录权限

* 其他问题：根据错误信息进行修复
