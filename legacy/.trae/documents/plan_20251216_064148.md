# user_rules.md文件优化计划

## 优化目标
将user_rules.md文件优化为符合个人规则配置引导的格式，明确TRAE在对话和代码生成时需要遵循的规则。

## 优化内容

### 1. 调整文件结构
- 简化文件头部说明，明确文件用途
- 保持规则分类清晰，易于理解

### 2. 完善规则内容
- 保留现有核心规则（语言要求、编码风格、注释规则、文件命名）
- 添加系统类型信息（Windows）
- 明确TRAe在对话和代码生成时的具体行为

### 3. 控制文件大小
- 保持简洁，确保字符数不超过1000
- 删除冗余内容，保留核心规则

### 4. 优化表达方式
- 使用清晰、直接的语言
- 采用列表形式，易于阅读

## 优化后的规则格式
```markdown
# 个人规则
定义TRAe在对话和代码生成时需要遵循的个人习惯和要求。

## 1. 对话规则
- 全程使用中文
- 输出内容简洁、清晰
- 系统环境：Windows

## 2. 编码风格
- 严格遵循PEP 8规范
- 使用4空格缩进，禁止Tab

## 3. 注释规则
- 每个函数必须添加docstring
- 关键步骤添加行内注释

## 4. 文件命名
- 统一使用小写字母+下划线格式（如：my_script.py）
```

## 预期成果
生成一份结构清晰、内容具体、符合要求的个人规则文件，帮助TRAe更好地理解和遵循用户的习惯和要求。
