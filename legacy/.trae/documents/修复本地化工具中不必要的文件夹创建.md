# 修复本地化工具中不必要的文件夹创建

## 问题分析

通过分析代码，我发现了问题所在：在`main.py`的`check_project_structure`函数中，代码直接在`source/English`和`source/Chinese`目录下创建了`src`和`jar`文件夹，这不符合文档要求。根据文档，这些文件夹应该是mod文件夹下的子目录，而不是直接出现在language目录下。

## 修复计划

### 1. 修复check_project_structure函数

**文件**：`src/main.py`
**修改内容**：
- 移除`localization_folders`列表中直接创建`src`和`jar`文件夹的路径
- 只创建到language级别（如source/English、source/Chinese）
- 让mod文件夹自行管理其内部的src和jar子目录

### 2. 具体修改点

将以下代码：
```python
# 定义 Localization_File 下的必要文件夹结构 - 严格按照框架文档
localization_folders = [
    # 源文件目录结构
    os.path.join(localization_file_path, "source"),
    os.path.join(localization_file_path, "source", "English"),
    os.path.join(localization_file_path, "source", "English", "src"),
    os.path.join(localization_file_path, "source", "English", "jar"),
    os.path.join(localization_file_path, "source", "Chinese"),
    os.path.join(localization_file_path, "source", "Chinese", "src"),
    os.path.join(localization_file_path, "source", "Chinese", "jar"),
    # 源文件备份目录结构
    os.path.join(localization_file_path, "source_backup"),
    os.path.join(localization_file_path, "source_backup", "English"),
    os.path.join(localization_file_path, "source_backup", "English", "src"),
    os.path.join(localization_file_path, "source_backup", "English", "jar"),
    os.path.join(localization_file_path, "source_backup", "Chinese"),
    os.path.join(localization_file_path, "source_backup", "Chinese", "src"),
    os.path.join(localization_file_path, "source_backup", "Chinese", "jar"),
    # ... 其他目录
]
```

修改为：
```python
# 定义 Localization_File 下的必要文件夹结构 - 严格按照框架文档
localization_folders = [
    # 源文件目录结构
    os.path.join(localization_file_path, "source"),
    os.path.join(localization_file_path, "source", "English"),
    os.path.join(localization_file_path, "source", "Chinese"),
    # 源文件备份目录结构
    os.path.join(localization_file_path, "source_backup"),
    os.path.join(localization_file_path, "source_backup", "English"),
    os.path.join(localization_file_path, "source_backup", "Chinese"),
    # ... 其他目录
]
```

### 3. 验证修复

修复后，工具将不再直接在language目录下创建src和jar文件夹，而是让mod文件夹自行管理其内部的src和jar子目录，这符合文档要求。

### 4. 测试计划

1. 运行工具，验证不再创建不必要的src和jar文件夹
2. 检查Localization_File目录结构，确保只有必要的目录存在
3. 执行Extract和Extend模式，验证工具仍能正常工作
4. 验证mod文件夹下的src和jar子目录能被正确处理
